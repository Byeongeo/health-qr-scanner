📷 QR 점수 스캐너 — 설치 가이드

학생이 QR(학번)을 카메라에 대면 점수가 쌓이고, 음성·화면으로 안내되는 우리 반 점수판을 만듭니다.

⏱️ 약 30~40분 💸 전부 무료 🧑‍💻 코딩 지식 필요 없음
준비물 ① 구글 계정(학교 계정 OK) · ② 크롬(Chrome) 브라우저 · ③ 웹캠 달린 데스크톱이나 태블릿 · ④ 학생 학번·이름 명단

✓ 전체 흐름 한눈에

  • 1단계 구글 시트 사본 만들기 (내 점수판 시트 생성)
  • 2단계 시트 속 Apps Script 웹앱 배포 → 내 주소 받기
  • 3단계 GitHub 계정 만들기 (처음이면 · 다음 단계 로그인용)
  • 4단계 Deploy 버튼으로 내 Vercel 앱 만들기 (주소 붙여넣기)
  • 5단계 학생 학번·이름 입력 → QR 자동 생성 → 인쇄
  • 6단계 교실 기기에서 열고 테스트

1 구글 시트 사본 만들기 약 1분

  1. 아래 버튼을 누릅니다. (연수 선생님이 준 링크입니다)
    📋 점수판 시트 사본 만들기
  2. 창이 뜨면 사본 만들기 를 누릅니다.
  3. 내 구글 드라이브에 “…의 사본” 시트가 생기고 자동으로 열립니다. 이게 내 점수판입니다.
참고 사본을 만들면 안에 들어있는 프로그램(코드)과 탭(student list 등)이 그대로 같이 복사됩니다. 따로 만들 게 없어요.

2 시트 속 프로그램을 “웹앱”으로 배포 약 5분 · 한 번만

  1. 방금 만든 사본 시트의 상단 메뉴에서 확장 프로그램 → Apps Script 를 누릅니다. (새 탭이 열립니다)
  2. 오른쪽 위 파란 버튼 배포 → 새 배포 를 누릅니다.
  3. 왼쪽 위 톱니바퀴(⚙️ 유형 선택)를 누르고 웹 앱 을 고릅니다.
  4. 설정을 이렇게 맞춥니다:
    • 설명: 아무거나 (예: QR스캐너)
    • 실행 계정(다음 사용자로 실행): 나
    • 액세스 권한(액세스 권한이 있는 사용자): 모든 사용자
  5. 배포 를 누릅니다.
  6. 권한 승인 창이 뜹니다 → 액세스 승인 → 내 구글 계정 선택.
  7. “Google에서 확인하지 않은 앱” 경고가 나오면(정상입니다, 내가 만든 거라 안전):
    → 왼쪽 아래 고급 → (프로젝트 이름)(으)로 이동 → 허용
  8. 완료되면 웹 앱 URL이 나옵니다. https://script.google.com/macros/s/……/exec 형태죠. 이 주소를 복사해서 메모장 등에 잠깐 붙여두세요. (3단계에서 씁니다)
꼭 확인! 액세스 권한이 “모든 사용자”여야 합니다. “모든 Google 사용자”나 “나만”으로 하면 앱이 작동하지 않습니다.
주소가 /exec 로 끝나야 함 — 주소 끝이 /dev가 아니라 /exec인지 확인하세요.

3 GitHub 계정 만들기 약 7분 · 처음이면

다음 4단계(Vercel)에서 로그인에 필요합니다. 이미 GitHub 계정이 있으면 이 단계는 건너뛰세요.

  1. 가입 페이지를 엽니다.
    GitHub 가입 페이지 열기 ↗
  2. 이메일 입력 → Continue.
    학교 이메일도 됩니다. 인증코드를 받아야 하니 그 메일함을 열 수 있어야 해요.
  3. 비밀번호를 만들고 → Continue.
  4. 사용자 이름(Username)을 정하고 → Continue.
    영문·숫자로, 남이 안 쓴 이름이어야 합니다(예: health-kim2026). 꼭 적어두세요 — 다음 단계에서 씁니다.
  5. 광고메일 수신 여부에 y 또는 n 입력 → Continue.
  6. “사람인지 확인” 퍼즐을 풉니다 → Create account.
  7. 입력한 이메일로 온 인증 코드(숫자)를 입력합니다.
  8. 가입 직후 설문/요금제 화면이 나오면 무료(Free)를 고르거나 Skip 으로 건너뜁니다. 완료! 🎉
Vercel은 따로 가입하지 않아도 됩니다. 다음 단계에서 “Continue with GitHub”를 누르면 이 GitHub 계정으로 Vercel까지 자동 로그인돼요.

4 내 Vercel 앱 만들기 (Deploy 버튼) 약 8분 · 한 번만

  1. 아래 Deploy 버튼을 누릅니다.
  2. Vercel 로그인 화면 → Continue with GitHub (3단계에서 만든 GitHub 계정으로 로그인). GitHub가 “Authorize Vercel” 또는 “Install” 을 물으면 눌러서 허용합니다.
  3. “Create Git Repository” 단계 → GitHub 선택하고 그대로 Create. (내 GitHub에 코드 사본이 하나 생깁니다 — 신경 안 써도 됩니다)
  4. 환경 변수(Environment Variables) 입력 칸이 나옵니다. ⭐가장 중요⭐
    • Key(이름): APPS_SCRIPT_URL (이미 채워져 있을 수 있음)
    • Value(값): 2단계에서 복사한 /exec 주소를 붙여넣기
  5. Deploy 를 누르고 1~2분 기다립니다.
  6. “Congratulations!” 화면이 뜨면 끝! Continue to Dashboard 또는 미리보기를 눌러 내 앱 주소(예: health-qr-scanner-xxxx.vercel.app)를 확인합니다. 이 주소를 교실 기기에서 즐겨찾기 해두세요.
환경 변수를 빼먹으면 스캔할 때 “서버에 연결하지 못했습니다”가 뜹니다. 그땐 Vercel → 내 프로젝트 → Settings → Environment Variables 에서 APPS_SCRIPT_URL 추가 후 Deployments에서 Redeploy 하면 됩니다.

5 학생 등록 + QR 인쇄 약 5분

  1. 1단계의 내 점수판 시트로 돌아갑니다.
  2. student list 탭에서 예시 2명(홍길동·김보건)을 지우고, A열에 학번 · B열에 이름을 입력합니다.
  3. 학번을 입력하면 C열에 QR이 자동으로 생깁니다. (외부 사이트 필요 없음)
  4. C열을 인쇄해서 학생별로 잘라 나눠주거나, 학생증/이름표에 붙입니다.
QR 안에는 “학번”만 들어갑니다. 그래서 학생이 그 QR을 카메라에 대면 누구인지 바로 인식돼요.

6 교실에서 사용·테스트 약 3분

  1. 웹캠 달린 기기의 크롬에서 내 Vercel 앱 주소를 엽니다.
  2. 화면을 한 번 클릭(=나중에 풋마우스 밟기) → 카메라 권한 “허용”.
  3. 학생 QR(또는 시트 C열 QR을 휴대폰에 띄워)을 카메라에 댑니다.
  4. “삑” 소리 + 음성 “○○○ 학생, 10점이 가산되어 총 △△점입니다” + 화면에 크게 표시되면 성공! 🎉
풋마우스(발판) 연결법 — 발로 밟아 카메라를 켜려면: ① 발판을 연결하고 메모장을 열어 한 번 밟아보세요. 글자가 찍히면(예: b) 그 글자를 시트 config 탭의 트리거키에 입력하고 스캐너 화면을 새로고침하면 끝. (영문 기준으로 적으세요 — 한/영 상태는 앱이 알아서 처리합니다) ② 아무 글자도 안 찍히는 발판은 "마우스 클릭"형입니다 — 트리거키와 상관없이 그냥 작동해요. 결과 화면에서 또 밟으면 바로 다음 학생 카메라가 켜집니다.

⚙ 점수·멘트 바꾸기 선택

모두 시트에서 바꿉니다. 코드는 건드릴 필요 없습니다.

  1. config 탭 — 회당가점(1회 몇 점), 쿨다운초(연속 적립 방지), 미등록처리 등.
  2. messages 탭 — 점수대별 음성 멘트.
    • 임계점수: 이 점수 이상일 때
    • 멘트: 들려줄 말 ({이름} 자리에 학생 이름이 들어감)
    • 1회만: Y면 그 점수를 처음 넘을 때 한 번만(예: “선물 받아가세요”), N이면 그 점수대에서 매번(예: “잘했어요”)

？ 문제 해결

소리가 안 나요
기기 음량·브라우저 탭 음소거를 확인하세요. 크롬을 권장합니다(한국어 음성이 가장 잘 나옵니다).
카메라가 안 켜져요
주소창 오른쪽 카메라 아이콘을 눌러 “허용”으로 바꾸세요. 그리고 화면을 한 번 클릭(밟기)해야 켜집니다.
“서버에 연결하지 못했습니다”
3단계 환경 변수(APPS_SCRIPT_URL)가 빠졌거나 주소가 틀렸습니다. Vercel Settings에서 확인 후 Redeploy 하세요.
“등록되지 않은 학번입니다”
그 QR의 학번이 student list A열에 없습니다. 학번을 추가하거나 철자를 확인하세요.
탭(student list 등)이 안 보여요
시트를 새로고침한 뒤 상단 메뉴 QR 점수 스캐너 → 초기 설정 을 한 번 누르세요. 탭이 자동으로 만들어집니다.
“Apps Script 응답을 해석할 수 없습니다”
2단계 배포의 액세스 권한이 “모든 사용자”인지, 주소가 /exec로 끝나는지 확인하세요.
